About the Project

Affirm is a web application being built by the Trans Equity Coalition (TEC), a non-profit organisation. The project is in its early stages — the technical scaffolding and infrastructure are in place, but the substantive frontend work is just beginning. This is a greenfield opportunity to shape the application from the ground up.

The codebase is hosted on GitHub within a multi-repo organisation. The main application repo (affirm) is where the vast majority of frontend work will happen. Supporting repos handle org-level configuration, internal discussions, presentations, and encrypted secrets management.

The team is small. There is currently a primary maintainer handling architecture, DevOps, and backend concerns, with at least one other contributor who has begun initial work. The frontend lead role is a critical gap - the person filling it will have significant influence over the direction and quality of the user-facing application.

Tech Stack Overview

Nuxt 4 + Vue 3

Full-stack Vue framework with file-based routing, auto-imports, server routes, and the Composition API via <script setup lang="ts">.

Cloudflare Workers

Edge deployment using the cloudflare_module Nitro preset. Fast cold starts, global distribution, no traditional server to manage.

Cloudflare D1

SQLite-based serverless database with Drizzle ORM for type-safe schema definition, migrations, and queries.

Tailwind CSS v4 + DaisyUI v5

Utility-first CSS with a comprehensive component library. Themed with the Catppuccin Mocha colour palette for a distinctive dark-mode aesthetic.

TypeScript

Strict typing throughout - components, composables, server routes, and database schema are all fully typed.

Bun

Fast JavaScript runtime and package manager (v1.3.10). Used for installs, scripts, and the dev server.

Additional Nuxt Modules

The project has nine Nuxt modules pre-configured, covering accessibility (@nuxt/a11y), content management (@nuxt/content), icon rendering, image optimisation, font handling, ESLint integration, performance hints, and script management. This is a well-stocked foundation - the tooling decisions have been made so you can focus on building.

What the Frontend Lead Will Do

This is a hands-on leadership role. You'll be the primary person building the user interface while also setting standards for any contributors who join later.

Build the UI

Translate designs and product requirements into Vue 3 components using the Composition API, Tailwind CSS, and DaisyUI. This is the core of the role - building pages, forms, data displays, and interactive features.

Establish Patterns

Define component architecture, composable patterns, state management approach, and naming conventions. You'll set the precedent that future contributors follow.

Own Accessibility

The @nuxt/a11y module is already in place. Ensure the application meets WCAG standards - semantic HTML, keyboard navigation, screen reader support, and colour contrast.

Write Tests

Establish the testing culture. Vitest for component/unit tests and Playwright for end-to-end flows. Both are installed and configured, just waiting for tests.

Collaborate on API Design

Work with the backend/infra maintainer to define API contracts. Nuxt's server routes live in the same repo, so you'll have full visibility into the data layer.

Performance

Leverage Nuxt's SSR on Cloudflare Workers for fast initial loads. Optimise bundle size, lazy-load where appropriate, and make use of the @nuxt/hints and @nuxt/image modules.

Coding Patterns & Conventions

Components

Vue 3 Single File Components using <script setup lang="ts">. All components in app/components/ are auto-imported - no manual import statements needed. Props are typed with TypeScript interfaces.

Composables

Reactive utilities in app/composables/ follow the use* naming convention and are auto-imported. These are the building blocks for shared state and logic.

Server Routes

API endpoints live in server/api/ using Nitro's file-based routing. They follow the name.method.ts convention (e.g. hello.get.ts). D1 access uses a useDB(event) helper.

Shared Utilities

Code in shared/utils/ is auto-imported in both the app and server contexts. Ideal for pure functions like formatting, validation, and type guards.

Style Conventions

Indentation is 2 spaces for TypeScript/Vue/JSON (4 spaces for Python, Rust, and Fish; tabs for Go). Max line length is 120 characters. Trunk handles all linting and formatting enforcement, and CI will catch any violations.
